This visual guide has been created to display the compliance with WCAG accessibility standards for color contrast. To check the colors yourself, you can use this link: https://webaim.org/resources/contrastchecker/

Color contrast is the difference in light between font (or anything in the foreground) and its background. In web accessibility, how well one color stands out from another color determines whether or not most people will be able to read the information.[1]

WCAG 2.0 level AA requires a contrast ratio of at least 4.5:1 for normal text and 3:1 for large text. WCAG 2.1 requires a contrast ratio of at least 3:1 for graphics and user interface components (such as form input borders). WCAG Level AAA requires a contrast ratio of at least 7:1 for normal text and 4.5:1 for large text.

Large text is defined as 14 point (typically 18.66px) and bold or larger, or 18 point (typically 24px) or larger.[2]

Buttons

Use the dark turquoise over lighter background colors to give visual contrast for the button style against the background. Use a dark enough turquoise so that white text fits well over the button.


Use the dark turquoise as both the button outline and the text color, and transparent for the button fill. Put this button style over a background color that is light enough that both draw enough visual attention through contrast, but not enough to distract from the primary CTA.


We also recommend using a text link style for any longer calls to action to make sure the information seems clickable. We've added an arrow style after the link text to ensure that it is clear that the element is clickable.
